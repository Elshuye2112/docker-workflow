From Manual Setup to Docker

This demo shows how a simple Node.js app is deployed the traditional way vs. inside a Docker container — and why containers make life easier.

Before Docker Traditional

  • Install Node.js manually on every machine
  • Match exact Node.js version across environments
  • Run node app.js directly on the host OS
  • Works on your machine, may break on others
  • Manual steps to set up each new server
  • No isolation between apps on the same server

With Docker Containerized

  • One Dockerfile defines the entire environment
  • Same image runs identically everywhere
  • No Node.js needed on the host — just Docker
  • Isolated from other apps on the same machine
  • Deploy with a single docker run command
  • Push to Docker Hub — pull and run anywhere

Docker Workflow — Step by Step

1

Build the image

docker build -t berhanbank-app .
2

Run locally

docker run -p 3000:3000 berhanbank-app
3

Tag & push to Docker Hub

docker push yourusername/berhanbank-app
4

Pull & run anywhere

docker pull yourusername/berhanbank-app

Docker Commands Reference

Images
docker build -t my-app .
Build an image from a Dockerfile in the current directory
docker images
List all locally available images
docker rmi my-app
Delete a local image by name or ID
docker tag my-app user/my-app:v1
Tag an image before pushing to a registry
Containers
docker run -d -p 4000:4000 my-app
Run a container in the background and map port 4000
docker ps
List currently running containers
docker ps -a
List all containers including stopped ones
docker stop <container>
Gracefully stop a running container
docker rm <container>
Remove a stopped container
docker rm -f <container>
Force-remove a running container immediately
Logs & Debugging
docker logs <container>
Print the stdout/stderr logs of a container
docker logs -f <container>
Follow (tail) live logs from a running container
docker exec -it <container> sh
Open an interactive shell inside a running container
docker inspect <container>
Show detailed low-level info about a container or image
Registry
docker login
Authenticate to Docker Hub (or a private registry)
docker push user/my-app:v1
Upload a tagged image to Docker Hub
docker pull user/my-app:v1
Download an image from Docker Hub
Cleanup
docker system prune
Remove all stopped containers, dangling images, and unused networks
docker volume ls
List all Docker volumes on this machine
docker network ls
List all Docker networks
